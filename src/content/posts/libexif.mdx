## socket套接字---->(`TCP` and `UDP`)

套接字是网络编程中的一种通信机制，是支持`TCP/IP`的网络通信的基本操作单元，可以看做是不同主机之间的进程进行双向通信的端点，简单的说就是通信的两方的一种约定，用套接字中的相关函数来完成通信过程（这个是概念，了解一下即可）

**IP**:

- `IP`主要的作用是：标识网络中不同的主机地址，
- `ip`分为`根源IP`和`目的IP`，简单理解，根源`IP`用于确定你的大致范围，目的`IP`为了确定你的实际位置
- IPv4（默认情况是IPv4）IP地址是4个字节，32位整数，点分十进制表示IP地址，例如：127.0.0.1;用点分割每一个数字表示一个字节，每个字节的范围是0~0xff,也就是0~256

**端口号**：

端口号（port）是传输层协议的内容。

- 端口号用来标识一个进程，告诉操作系统，当前这个数据交给哪一个程序进行解析；
- IP地址 + 端口号能标识网络上的某一台主机的某一个进程；
- 一个端口号只能被一个进程占用。
- 端口号是一个2字节16位的整数；

**socket**函数：

```c
//创建socket文件描述符  (TCP/UDP,客户端+服务器)
int socket(int domain, int type, int protocol);
```

```bash
domain (协议族):
宏定义     对应数值   用途
AF_UNIX / AF_LOCAL 1    本地进程间通信 (Unix Domain)
AF_INET    2       IPv4 网络协议
AF_INET6   10       IPv6 网络协议 (在不同架构下可能有差异)
AF_ROUTE   16       路由套接口 (Linux 下常用 AF_NETLINK)
AF_KEY    15    密钥管理套接口
```

```bash
Type (套接字类型)
宏定义     对应数值 描述
SOCK_STREAM 1 面向连接的流（TCP）
SOCK_DGRAM 2 无连接的数据报（UDP）
SOCK_RAW 3 原始套接字（允许访问底层协议）
```

```bash
Protocol (特定协议)

对于 protocol 参数，大多数情况下我们传入 0（对应你提到的 NULL，但在 C 语言中通常直接写 0）。
0 (IPPROTO_IP)：让内核根据 type 自动选择默认协议。比如 AF_INET + SOCK_STREAM 组合下，系统会自动识别并使用 TCP(6)
IPPROTO_TCP (6)：明确指定使用 TCP。
IPPROTO_UDP (17)：明确指定使用 UDP。
```

**bind**函数：

```c
//绑定端口号  （TCP/IP，服务器）
int bind(int socket, const struct sockaddr *address, socklen_t address_len);
```

这里的一参是`socket`返回的文件描述符，二参是`struct sockaddr *`类型的结构体，但是一般我们会选择先创建一个`struct sockaddr_in` 结构体然后强转，

一般的`struct sockaddr_in` 结构体具备下面的内容：

```c++
sin_family: 协议族（如 AF_INET）。
sin_addr: IP 地址（通常用 INADDR_ANY 表示监听所有网卡，或者指定特定 IP）。
sin_port: 端口号（必须使用 htons() 转换为网络字节序）。
```

然后由于`bind` 支持多种协议（IPv4, IPv6 等），内核需要通过这个长度来确定如何解析参数 2，

示例：

```c
struct sockaddr_in serv_addr;
memset(&serv_addr, 0, sizeof(serv_addr)); // 初始化

serv_addr.sin_family = AF_INET;           // 使用IPv4
serv_addr.sin_addr.s_addr = htonl(INADDR_ANY); // 监听所有网卡
serv_addr.sin_port = htons(8080);         // 绑定8080端口

if (bind(sockfd, (struct sockaddr*)&serv_addr, sizeof(serv_addr)) == -1) {
    perror("bind failed");
    exit(1);
}
```

**listen**函数

```c
// 开始监听socket  (TCP,服务器）
int listen(int socket, int backlog);
```

- 参数1(sockfd)：是由socket()调用返回的并且未作连接的套接字描述符（套接字号）

- 参数2(backlog)：\*\*所监听的端口队列大小

  这里需要解释一下**backlog (积压队列大小)**，它**不是**服务器能处理的最大并发连接数，而是**已完成握手但尚未被 `accept` 提取的连接队列**的最大长度。
  - 当客户端发起三次握手时，连接会进入内核的队列中
  - 如果你的程序忙于处理其他逻辑，还没来得及调用 `accept()`，这些连接就会在 `backlog` 队列里排队
  - **如果队列满了：** 客户端可能会收到 `ECONNREFUSED` 错误（连接被拒绝）

**accept**函数

```c
//接受请求  （TCP，服务器）
int accept(int socket, struct sockaddr* address, socklen_t* address_len);
```

当程序调用`accept`时，accept会返回一个套接字，后续将会对这个套接字进行read/write,而原先有socket创建的一个套接字将会继续回到 `listen` 状态，等待下一个新人(请求)的到来,

参数 2：address (客户端地址信息)

这是一个**输出参数**。当你调用 `accept` 时，内核会把**客户端**的 IP 地址和端口号填进这个结构体里。

- 如果你不关心客户端是谁（比如匿名服务），可以传 `NULL`。

参数 3：address_len (地址长度指针)

这也是一个**输出参数**（注意它是一个指针 `socklen_t*`）。

- **调用前：** 你告诉内核你的 `address` 缓冲区有多大。
- **调用后：** 内核告诉你它实际写入了多少字节。

`accept()` 是一个**阻塞函数**。如果 `backlog` 队列里没有客户端在排队，程序会停在这里“睡觉”，直到有客户端完成三次握手

`connect`和`close`

```c
//建立连接  （TCP，客户端）
int connect(int sockfd, const struct struct sockaddr *addr, aocklen_t addrlen);.

//关闭套接字
int close(int fd);
```

**参数(fd)：**是由socket()调用返回的并且未作连接的套接字描述符（套接字号）。

socket API是一层抽象的网络编程接口，适用于各种底层网络协议，如IPv4，IPv6，……

那么三次握手 ，指的是什么呢？

```bash
SYN：客户端发送“我想连接”。
SYN-ACK：服务器回应“收到，我准备好了”。
ACK：客户端最后确认“收到，开始聊天”。
注意： 客户端在调用 connect 之前通常不需要调用 bind。操作系统会自动为客户端分配一个可用的临时端口（Ephemeral Port）。
```

最后第四次握手：`通知对方：“我没有数据要发送了。”；等待对方确认并关闭其发送端。`
